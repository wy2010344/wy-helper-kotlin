# 基于信号 MVVM 渲染框架的设计模式

本文档总结本框架中约定俗成的编码模式，聚焦于**匿名类 + 属性重载**与**信号驱动状态**两个核心范式。

---

## 1. 匿名类与属性重载

**原则**：不通过构造函数参数传递配置，而是定义 `open val` 属性，使用者通过匿名类 `override` 来定制。

```kotlin
// ✅ 推荐：open 属性 + 匿名类 override
open class TextNode(context: StateHolder<Node>) : RectNode(context) {
    open val text: String = ""
    open val fontSize: Float = 16f
    open val color: ColorInt = rgba(0, 0, 0)
}

// 使用时：
object : TextNode(this) {
    override val text: String
        get() = mySignal.value
    override val fontSize: Float
        get() = 20f
}
```

### 优点

- **就近自定义**：所有配置集中在一个匿名类体内，无需翻阅构造参数列表
- **可重写 Getter**：属性可用 `get()` 从信号中实时读取，自动参与反应式追踪
- **继承友好**：子类只需 `override` 需要的属性，其他用默认值
- **零样板**：无需手动声明 setter 或额外配置函数

### 例外场景

某些需要运行时变化且非信号驱动的参数（如尺寸约束），仍需覆盖方法而非属性：

```kotlin
object : RectNode(this) {
    override fun size(direction: Direction): LayoutSize {
        return LayoutSize(100f, false)
    }
}
```

## 2. 信号驱动的状态变化

**原则**：所有可变状态都必须使用 `by createSignal(...)` 属性委托，赋值即触发反应式更新。

```kotlin
class TextNode(context: StateHolder<Node>) : RectNode(context) {
    private var anchorIndex by createSignal(-1)
    private var focusIndex by createSignal(-1)

    fun updateSelection() {
        anchorIndex = 5    // 触发信号，通知所有依赖
        focusIndex = 10    // 同上
    }
}
```

### 原理

- `createSignal(initial)` 返回 `StoreRef<T>`，支持 `getValue/setValue` 委托
- 读取时自动注册当前 `TrackSignal`/`Memo` 为依赖
- 写入时若值变化，自动标记状态版本、加入批处理队列、触发重渲染
- `memo { ... }` 内读取的信号会被追踪，信号变化时 memo 自动失效

### 哪些必须用信号

| 场景 | 信号 | 原因 |
|---|---|---|
| 界面状态（选中、展开） | `by createSignal(...)` | 改变需触发重绘 |
| 简单局部变量（循环计数） | 普通变量 | 无需反应式 |
| 属性 `get()` 中返回信号 | `override val x get() = signal.value` | 每次读取时追踪 |

## 3. 节点间通信：全局事件总线

### 问题

文本拖拽选中时，鼠标可能移出 TextNode 边界。此时无法通过命中测试发送事件。因此需要一套**与命中测试解耦**的全局事件机制。

### 方案：MVVM Context + 全局注册

```
Renderer (provides EngineGlobal)
   └─ provide(engineGlobalContext, impl)
        ├─ TextNode.registerMouseMove → 鼠标移动时收到全局坐标
        ├─ TextNode.registerMouseUp   → 鼠标释放时收到全局坐标
        └─ 其他节点同理
```

```kotlin
// ————— 定义 (EngineGlobal.kt) —————
typealias MouseCallback = (x: Float, y: Float) -> Unit

interface EngineGlobal {
    fun registerMouseMove(callback: MouseCallback): EmptyFun
    fun registerMouseUp(callback: MouseCallback): EmptyFun
}

val engineGlobalContext = Context<EngineGlobal?>(null)
```

```kotlin
// ————— 提供 (Renderer.kt) —————
private val moveList = mutableSetOf<MouseCallback>()
private val upList = mutableSetOf<MouseCallback>()

private val state = renderRoot<Node>(this@Renderer, ::collectIndex) {
    provide(engineGlobalContext, object : EngineGlobal {
        override fun registerMouseMove(callback: MouseCallback): EmptyFun {
            moveList.add(callback)
            return { moveList.remove(callback) }
        }
        override fun registerMouseUp(callback: MouseCallback): EmptyFun {
            upList.add(callback)
            return { upList.remove(callback) }
        }
    })
    buildChildren()
}
```

```kotlin
// ————— 消费 (TextNode.init) —————
init {
    val engineGlobal = context.consume(engineGlobalContext)!!
    val d1 = engineGlobal.registerMouseUp { x, y ->
        onMouseDown = false
    }
    val d2 = engineGlobal.registerMouseMove { x, y ->
        if (onMouseDown) focusIndex = charAt(x - absoluteX())
    }
    context.addDestroy { d1(); d2() }
}
```

### 在平台上转发事件

```kotlin
// Desktop: SkiaWindow.desktop.kt
skiaLayer.addMouseMotionListener(object : MouseMotionAdapter() {
    override fun mouseMoved(e: MouseEvent?) {
        this@SkiaApp.mouseMove(e.x.toFloat(), e.y.toFloat())
    }
    override fun mouseDragged(e: MouseEvent?) {
        this@SkiaApp.mouseMove(e.x.toFloat(), e.y.toFloat())
    }
})
```

Renderer 的 `mouseMove` 直接遍历并调用所有注册的回调：

```kotlin
fun mouseMove(x: Float, y: Float) {
    moveList.forEach { it(x, y) }
}
```

## 4. 坐标转换体系

### 本地坐标 → 绝对坐标

```kotlin
fun Node.absolutePosition(d: Direction): Float {
    var n = position(d)
    var p = parent
    while (p != null) {
        n += p.position(d)
        p = p.parent
    }
    return n
}
```

### 全局坐标 → 本地坐标

```kotlin
// Renderer 中的 drag 处理
private fun rootToLocal(node: Node, rootX: Float, rootY: Float): Pair<Float, Float> {
    var x = rootX
    var y = rootY
    var n: Node? = node
    while (n != null) {
        x -= n.position(Direction.x)
        y -= n.position(Direction.y)
        n = n.parent
    }
    return Pair(x, y)
}
```

在全局鼠标回调中，全局坐标需要减去 `absoluteX()` 才得到 TextNode 本地坐标：

```kotlin
val absoluteX = memo { absolutePosition(Direction.x) }
engineGlobal.registerMouseMove { x, y ->
    if (onMouseDown) focusIndex = charAt(x - absoluteX())
}
```

## 5. TextNode 文字选择完整流程

```
用户操作             系统响应
─────────────────────────────────────────────────
1. 鼠标按下           mouseDown(e)
                     ↓
                     anchorIndex = charAt(e.x)
                     focusIndex  = anchorIndex
                     onMouseDown = true

2. 鼠标拖动（移出边界） mouseMove 全局事件
                     ↓
                     focusIndex = charAt(x - absoluteX())
                     选区更新 → drawSelf 重绘高亮

3. 鼠标释放           mouseUp 全局事件
                     ↓
                     onMouseDown = false
                     选区冻结

4. 获取选中文字       selectionText
                     ↓
                     text.substring(min, max)
```

### 字符位置计算

```kotlin
private val charPositions = memo {
    if (text.isEmpty()) return@memo listOf(0f)
    val positions = mutableListOf<Float>()
    for (i in text.indices) {
        positions.add(PlatformCanvas.measureText(
            text.substring(0, i + 1), fontFamily, fontWeight, fontSize
        ))
    }
    positions
}

private fun charAt(x: Float): Int {
    val positions = charPositions()
    for (i in positions.indices) {
        if (positions[i] > x) return i
    }
    return positions.size
}
```

### 选中高亮绘制

```kotlin
override fun drawSelf(canvas: PlatformCanvas) {
    if (text.isEmpty()) return
    val positions = charPositions()
    if (anchorIndex >= 0 && focusIndex >= 0 && anchorIndex != focusIndex) {
        val start  = min(anchorIndex, focusIndex)
        val end    = max(anchorIndex, focusIndex)
        val leftX  = if (start == 0) 0f else positions[start - 1]
        val rightX = positions[end - 1]
        canvas.drawRect(x = leftX, w = rightX - leftX, h = lightHeight, color = selectionColor)
    }
    canvas.drawText(text, 0f, fontSize, fontFamily, fontWeight, fontSize = fontSize, color = color)
}
```

## 6. 生命周期管理

| 钩子 | 用法 |
|---|---|
| `init` 块 | 注册全局事件、初始化 memo |
| `context.addDestroy { ... }` | 清理全局回调、释放资源 |
| `addDestroy { ... }` (在 `buildChildren` 中) | 清理 `renderForEach` 创建的节点 |
| `destroyed` 属性 | 检查节点是否已被销毁 |

```kotlin
init {
    val d1 = engineGlobal.registerMouseMove { ... }
    val d2 = engineGlobal.registerMouseUp { ... }
    context.addDestroy {
        d1()  // 反注册 move
        d2()  // 反注册 up
    }
}
```

## 7. WrappedTextNode：文字换行与多行选择

WrappedTextNode 专用于**固定宽度容器内文字换行展示与选择**，继承自 RectNode，遵循相同的匿名类 + 属性重载模式。

### 核心属性

```kotlin
open class WrappedTextNode(context: StateHolder<Node>) : RectNode(context) {
    open val text: String = ""
    open val fontSize: Float = 16f
    open val fontWeight: Int = 400
    open val fontFamily: String? = null
    open val lineHeight: Float get() = fontSize * 1.4f
    open val color: ColorInt = rgba(0, 0, 0)
    open val selectionColor: ColorInt = rgba(0, 100, 200, 60)
    open val wrappingWidth: Float = Float.MAX_VALUE  // 换行宽度
}
```

- `wrappingWidth`：换行边界宽度。默认 `Float.MAX_VALUE`（不换行）。设为具体值（如 `350f`）即开启换行
- `lineHeight`：行高，默认 `fontSize * 1.4f`

### 换行算法

算法分三步（借鉴 GUI 框架常见策略）：

```
1. 二分查找在当前行能容纳的最长子串（字符级别贪心）
2. 若溢出点后有更多文字，尝试在最后一个空格处断行（word-break）
3. 若无空格可断，退化到字符级断行（overflow-wrap: break-word）

"Hello World Foo" 在 80px 宽度下：
  ┌─────────────────────────────────┐
  │ Line 0: "Hello"   (word break)  │
  │ Line 1: "World"   (word break)  │
  │ Line 2: "Foo"                   │
  └─────────────────────────────────┘
```

代码实现：

```kotlin
private val lines = memo {
    if (wrappingWidth >= Float.MAX_VALUE) return@memo listOf(TextLine(0, text.length))
    val result = mutableListOf<TextLine>()
    var start = 0
    while (start < text.length) {
        while (start < text.length && text[start] == ' ') start++
        if (start >= text.length) break

        var low = start
        var high = text.length
        while (low < high) {  // 二分查找最大适配前缀
            val mid = (low + high + 1) / 2
            val w = PlatformCanvas.measureText(text.substring(start, mid), ...)
            if (w <= wrappingWidth) low = mid else high = mid - 1
        }
        if (low == start) low = start + 1  // 至少一个字符

        val breakPos = if (low < text.length) {
            val spaceIdx = text.lastIndexOf(' ', low - 1)
            if (spaceIdx >= start) spaceIdx else low
        } else text.length

        result.add(TextLine(start, breakPos))
        start = if (breakPos < text.length && text[breakPos] == ' ') breakPos + 1 else breakPos
    }
    result
}
```

### 尺寸报告

```kotlin
override fun size(direction: Direction): LayoutSize {
    val lineList = lines()
    return when (direction) {
        Direction.x -> if (wrappingWidth < Float.MAX_VALUE)
            LayoutSize(wrappingWidth, false)   // 固定宽度，由用户/容器确定
        else
            LayoutSize(naturalWidth, true)     // 不换行时取实际文字宽度
        Direction.y -> LayoutSize(lineList.size * lineHeight, true)  // 高度由行数决定
    }
}
```

### 多行选择命中测试

`charAt(px, py)` 接收本地坐标，先通过 Y 坐标定位到行，再通过 X 坐标定位到字符：

```kotlin
private fun charAt(px: Float, py: Float): Int {
    val lineList = lines()
    val lineIdx = (py / lineHeight).toInt().coerceIn(0, lineList.size - 1)
    val line = lineList[lineIdx]

    var low = line.start; var high = line.end
    while (low < high) {
        val mid = (low + high) / 2
        val w = PlatformCanvas.measureText(text.substring(line.start, mid + 1), ...)
        if (w <= px) low = mid + 1 else high = mid
    }
    return low.coerceIn(line.start, line.end)
}
```

拖拽时，全局鼠标事件的 `(x, y)` 通过 `absolutePosition` 转为本地坐标：

```kotlin
val absoluteX = memo { absolutePosition(Direction.x) }
val absoluteY = memo { absolutePosition(Direction.y) }
engineGlobal.registerMouseMove { x, y ->
    if (onMouseDown) focusIndex = charAt(x - absoluteX(), y - absoluteY())
}
```

### 多行高亮绘制

逐行计算选区与文字的重叠部分，每行各自绘制高亮矩形：

```kotlin
for ((li, line) in lineList.withIndex()) {
    val lineStart = max(selStart, line.start)
    val lineEnd = min(selEnd, line.end)
    if (lineStart < lineEnd) {
        val leftX  = measureText(text.substring(line.start, lineStart))
        val rightX = measureText(text.substring(line.start, lineEnd))
        canvas.drawRect(x = leftX, y = li * lineHeight,
                        w = rightX - leftX, h = lineHeight, color = selectionColor)
    }
}
// 逐行绘制文字
for ((li, line) in lineList.withIndex()) {
    canvas.drawText(text.substring(line.start, line.end),
        0f, li * lineHeight + fontSize, ...)
}
```

## 8. PlatformCanvas 伴生对象测量

`measureText` 作为 `companion object` 的静态方法，无需 Canvas 实例即可随时调用：

```kotlin
// 定义
expect class PlatformCanvas {

PlatformCanvas.measureText("hello", fontSize = 24f)
```

## 8. 事件系统总览

```
                          Renderer
                     ┌─────────────┐
                     │ mouseDown() │──→ hitTest → mouseEventOf → 节点 capture/冒泡分发
                     ├─────────────┤
                     │ mouseUp()   │──→ hitTest + upList 全局事件
                     ├─────────────┤
                     │ mouseMove() │──→ moveList 全局事件（无 hitTest，mouseMoved 和 mouseDragged 都走这里）
                     ├─────────────┤
                     │ mouseClick()│──→ hitTest → mouseEventOf
                     └─────────────┘
                           │
                     ┌─────┴─────┐
                     │  EngineGlobal  │
                     │ (通过 Context  │
                     │  注入到子树)   │
                     └─────┬─────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
         TextNode  WrappedTextNode  未来更多节点
```

- `mouseDown`/`mouseUp`/`mouseClick`：通过 hitTest 沿节点树 capture → 冒泡分发
- `mouseUp` 额外触发 `upList` 全局回调
- `mouseMove` 仅触发 `moveList` 全局回调，不经 hitTest
- **mouseMoved 和 mouseDragged 都映射到 mouseMove**：平台层（Desktop）将 AWT 的 `mouseMoved` 与 `mouseDragged` 统一转发为 `Renderer.mouseMove()`，简化了全局事件模型。`WrappedTextNode` 通过 `onMouseDown` 标记区分"悬停"与"拖拽"。
